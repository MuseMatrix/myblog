Css盒子模型：

所有的元素都是容器，也叫盒子
所有的元素都是盒子模型

行内块和块级可以包所有类型的元素
行内元素只能包行内元素（内联元素）

padding: 内边距
        内边距会增加盒子大小
        举个例子：
            padding:40px 30px 20px 10px;
                    上    右   下    左
            padding:40px 30px 20px;
                    上   左右  下
            padding:40px 30px;
                    上下  左右
            一个值表示四个方向
border:边框
        举个例子：
            border:3px solid red;
                   粗细  、 风格、  颜色
                风格：solid 实线
                      dotted 点线
                      dashed 虚线
        边框会增加盒子的大小
margin:外边距
        举个栗子：
            margin:40px 30px 20px 10px;
                    上   右   下   左
            margin:40px 30px 20px;
                    上  左右  下
            margin:40px 30px;
                   上下  左右
            一个值表示四个方向
        margin:外边距不能增加盒子的大小

            复合样式：有多个值的样式
            单样式：单个属性
盒子实际大小 = width/height   +   内边距  +   边框
盒子的实际占据位置大小 =  width/height   +   内边距  +   边框   + 外边距

当一个元素出现在另一个元素上面时，第一个元素的下外边距与第二个元素的上外边距会发生合并
如果不设置父div 的内边距和边框，那么内部 div 的上外边距将与外部 div 的上外边距合并（叠加）。

（父元素包含子元素）解决外边距合并的方法
    1：父容器：border：1px solid;（不能是0px）
    2：父容器：overflow: hidden;（或auto）
    3：父容器：padding：1px；（不能是0或auto，其他的任意数值都可以）
    4：父容器或子容器： float：left；（不能是auto）
    5：父容器或子容器：position：absolute；
    6：子容器：display：inline-block;(或是inline-table)
    7：相邻元素：任意一个添加float：left（不能是auto）

Css盒模型解析模式

box-sizing:盒模型解析模式
    content-box:标准盒模型(和测试2一样的计算)
            宽度和高度 之外绘制元素的内边距和边框
            width,height外绘制padding,border,盒子的大小变大
    border-box:怪异盒模型
            从已设定的宽度和高度分别减去边框和内边距才能得到内容的宽度和高度
            width,height内绘制padding,border,盒子大小不变
border-box(怪异盒模型)和content-box(标准盒模型)                                                                                            的区别在于，怪异盒模型padding和border不会让元素整体宽度增加，而标准盒模型padding和border会让元素的整体宽度增加
弹性盒模型

Css隐藏

visibility:hidden;属性指定元素是否可见，即使不可见也会占据元素的空间
display:none;元素隐藏消失(不会占据元素的空间)

Css元素类型

display: 设置元素显示的类型；none:隐藏、消失
block:块级;inline行内
inline-block:行内块

块级元素：

1、独占一行
2、默认宽度为父级宽度
3、支持外边距
4、支持内边距
5、支持    margin:auto;居中
6、任何元素都可以包住
div  p   ul  li  ol  dl  dt  dd 
不听话的块级：p，它不能包含任何块级

行内元素：本质上就是文字

    1、能和其他行内元素在一行
    2、宽高由内容撑开，不支持宽高的设置
    3、不支持上下的外边距,支持左右外边距
    4、支持左右的内边距，上下内边距只支持下内边距
    5、不支持 margin:auto;居中
    6、行内元素不能包块级
    行内元素的内边距对左右下起作用；
    行内元素的外边距对左右起作用；
span   em   strong    i   b   a  img
不听话的行内: a，a不能包含a
inline-block 行内块特点：本质上还是一个文字
    1、不支持margin:  auto; 居中
    2、支持给父级设置text-align，从而实现自身的一个对齐
    3、可以设置宽度和高度
    4、可以设置margin
    5、可以设置padding
    6、能和行内元素呆一块
text-align:设置文本的对齐方式
left     左边
right    右边
center   居中
行内元素和行内元素之间的空白距离都会压缩成一个空格
    去除空格方法：
        1、挨在一起
        2、注释去空格
        3、元素父级设置文字大小为0，自身再单独设置文字大小

选择器的优先级：

选择器个数不能超过5个，一般是3个最好
优先级：
    id       >      class     >    标签      >   *通配符
    钻石卡         黄金卡         白银卡         黄铜卡
    同级选择器：后面的优先级要比前面的优先级高
    优先级比法：
        比个数：首先比id个数，如果个数相同，
                比类选择器的个数，如果个数相同，
                比标签选择器的个数，如果个数相同，
                比先后顺序。
        id:不管多少个class比不过一个id
        class:不管多少个标签比不过一个class
    特殊情况：
        在IE里面,256个class可以干掉一个ID;
        256个标签可以干掉一个class
    行内样式、内部样式、外部样式的优先级
        行内样式 >  内部样式 == 外部样式
        link引入外部的css,还能引入其他的
        @import   只能引入css文件
            @import url("css/02.css");
            加载顺序在最后面
            只能写在样式里，内部样式或者外部样式
            一定要顶格写

    !important  无视优先级，是当前这一个属性优先级最高

css浮动：

float:设置元素的浮动
    left:左浮动；
    right:右浮动；
    作用：
        让元素并排在一起
        控制整个元素的水平方向
    特点：
    1、飞起来了
    2、在普通元素上面
    3、会挤压别人的文字
    4、会被别人占据位置
    5、半脱离文档流
    6、遇到父级的边缘会停止浮动
    7、造成父级的高度撑不开，也就是没有高度，父级塌陷
    8、宽高度默认都由内容撑开
    9、margin: 0 auto;失效
    10、浮动会将元素变成块级
    11、不会发生外边距合并
    12、父级宽度不够的时候，浮动元素会掉下去
    13、前面强壮的浮动的元素的高度会卡主后面的浮动元素
clear   清除浮动带来的父级高度塌陷影响
left
right  
both    清除两边的浮动
清除浮动的作用：让父级有高度
清除浮动：
        1、给父元素加高度   不适用
        2、overflow:hidden;     不适用
        3、在浮动元素的最后面加块级，加清除    不适用
        4、伪元素清除  :after    目前主流的
           例子：
           .clearfix:after{
                display:block;
                content:"";
                clear:both;
           }

文档流：元素空间。从左到右，从上到下
        在文档流的元素：块级、行内块、行内
        只有在文档流里的块级才会发生外边距合并

Css定位

position    设置定位
    static    不定位，默认值
    absolute    绝对定位
    relative    相对定位
方位值：
    top
    left
    right
    bottom
绝对定位的特性：
    1、默认内容撑开宽度
    2、完全脱离文档流
    3、不规定位置的时候，原地飞起
    4、在没有给参考对象前，它会以Body为参考对象
    5、任何定位属性都能做绝对定位的参考对象，static除外
    6、参考对象只能是绝对定位的直系长辈
    7、往上找参考对象
    8、让元素变成块级
相对定位的特性：
    1、不会改变元素的特性
    2、相对于自身原本的位置移动
    3、人走了家还在
    4、即使移动了，原本的位置空间依然保留
    通常我们用相对定位做绝对定位的参考对象
固定定位的特性：
    1、默认内容撑开宽度
    2、完全脱离文档流,不占据空间
    3、不规定位置的时候，原地飞起
    4、它的参考对象永远只是浏览器的窗口
    5、让元素变成块级，会改变元素的特性
绝对定位元素和固定定位的元素层级比浮动和在文档流里的元素都高
    z-index   设置层级
        auto    默认值
    后面的元素层级会比前面的高
    比同级之间的层级
    z-index层级只对定位元素生效

overflow:内容的溢出方式；

visible:默认值 内容可见
hidden:隐藏,并且其余内容是不可见的；
scroll:出现滚动条；
auto:根据内容来决定是否出现滚动条；
over-flow-x:处理x轴方向的超出情况；
over-flow-y:处理y轴方向的超出情况；
只给一个方向的时候,另外一个方向是自动;

Css背景样式

background(背景颜色,背景图片,背景图像平铺展示方式,背景定位,背景图像固定)
background-color:背景颜色;(十六进制(#ffffff),颜色三原色rgb(235,207,221),英文单词(red))默认值为transparent(透明)
background-image:背景图片;
background-repeat:背景图像平铺展示方式;
    默认为平铺，默认值为none
    repeat:图像将在垂直方向和水平方向重复平铺该图像;
    repeat-x:设置图像水平平铺;
    repeat-y:设置图像垂直平铺;
    no-repeat:设置图像不平铺展示(只渲染一次图像);
background-position:(x,y)背景定位;background-position:(x,y)如果只有一个起始值，那么默认值为center
background-size  x 宽 ,y 高  背景尺寸
    contain    让背景图片完全显示在盒子里面，但是会造成盒子有一部分空白
    cover       让背景图片完全填满在盒子里面，但是会造成图片有
                一部分显示不出来
方向值：
    left top         right top             center       top
    left center      right center          center       center
    left bottom      right bottom          center       bottom
百分比：左上角为(0,0)、右下角为100%,100%、如果仅给一个值，那么另一个值为50%;
像素和百分比类似 
background-attachment:背景图像固定；
    scroll:背景图像随页面滚动而滚动
    fixed:背景图片固定;
background-clip 背景显示区域的裁剪
    border-box
        从边框区域外裁剪背景
    padding-box
        从内边距区域外裁剪背景
    content-box
        从内容区域外裁剪背景
background-orign 设置背景定位的原点
    border-box
        以边框的左上角为原点
    padding-box
        以内边距区域的左上角为原点
    content-box
        以内容区域的左上角为原点

CSS3弹性盒模型

FLEX,采用Flex布局的元素，称为Flex(Flex container),简称"容器"。它的所有子元素自动称为容器成员，称为Flex项目(felx item),简称"项目"，任何一个容器都可以指定为Flex布局
    display:flex | inline-flex 定义弹性伸缩盒容器类型
        flex  将对象作为弹性伸缩盒显示
        inline-flex 将对象作为内联块级弹性伸缩盒显示
            注意：设置为flex布局以后，子元素的float、clear和vertical-align属性将失效

容器默认存在两根轴：
    水平的主轴和垂直的交叉轴。主轴的开始位置(与边框的交叉点)main-start,结束位置叫main end,交叉轴的开始位置叫cross start,结束位置叫cross end
    项目默认沿主轴排列。单个项目占据的主轴空间叫做main size,占据的交叉轴空间叫做cross size
容器的属性,以下6个属性设置在容器上
    flex-direction(排列盒子方向)
        row(默认)：横向从左到右排列(横向)
        row-resverse 对齐方向与row相反             (倒序排列)
        column：纵向从上往下排列(纵向)。
        column-resverse:对齐方式与column相反       (倒序排列)
    flex-wrap(nowrap | wrap | wrap-reverse)
        控制flex容器是单行或者多行，同时横轴的方向决定了新行的堆叠方向
        nowrap(默认) flex容器为单行，不换行
        wrap:flex容器为多行。超出换行
        wrap-reverse: 反转wrap排列
    flex-flow
        属性是flex-direction属性和flex-wrap属性的简写形式，默认值为(row  nowrap)
    justify-content
        定义了项目在主轴上的对齐方向
        justify-content:flex-start | flex-end | center |space-between | space-around
        flex-start(默认值)：左对齐
        flex-end:右对齐
        center:居中
        space-between:两端对齐，项目之间的间隔都相等
        space-around:每个项目两侧的间隔相等。所以，项目之间的间隔比项目与边框的间隔大一倍
    align-items
        定义了项目在交叉轴上的如何对齐
        align-items:flex-start | flex-end |center | baseline | stretch;
        具体的对齐方式与交叉轴的方向有关，下面假设交叉轴从上到下
        flex-start:交叉轴的起点对齐。
        flex-end:交叉轴的终点对齐。
        center：交叉轴的中点对齐
        baseline:项目第一行文字的基线对齐;
        stretch(默认值)：如果项目未设置宽高或设为zuto,将沿交叉轴沾满整个容器的高度或宽度。
    align-content
        属性定义了多根轴线的对齐方式。如果项目只有一根轴线，该属性不起作用
        align-content:flex-start | flex-end | center | space-between |space-around |stretch;
        flex-start: 与交叉轴的起点对齐。
        flex-end:与交叉轴的终点对齐
        center:与交叉轴的中点对齐

        space-between:与交叉轴两端对齐，轴线之间的间隔平均分布。
        space-around:每根轴线两侧的间隔都相等。所以，轴线之间的间隔比轴线与边框的间隔大一倍
        stretch(默认值)：各行将会拉伸以占用剩余的空间。如果剩余的空间是负数，改值等效于‘flex-start’。在其他情况下，剩余空间被所有行平分，以扩大他们的侧轴尺寸
        多行下的align-content使align-items,两者之间会有冲突，所以看情况使用

    以下6个属性设置在项目上：
        order 项目排序
            定义项目的排列顺序。数值越小，排列越靠前，默认为0，可负值
        flex-grow  项目拓展比例
            <number>定义项目的放大比例，(默认为0) 即如果存在剩余空间，也不放大。
                子元素的尺寸 =  父级盒子剩余空间尺寸*子元素的flex-grow属性值/所有子元素的flex-grow属性值的和
        flex-shrink  项目缩小比例
            定义项目缩小比例，(默认为1)，即如果空间不足，该项目将缩小。为0就不缩小
            元素收缩宽度 = 超出的宽度 * 元素收缩比/收缩总比

            如果一个项目的flex-shrink属性值为0，其他项目都为1，则空间不足时，前者不缩小。负值对该属性无效。
        flex-basis  定义拓展剩余空间前，的交叉轴的宽
            定义了项目再分配多余空间之前，项目占据主轴空间(main size)浏览器根据这个属性，计算主轴是否有多余空间。它的默认值为auto.
            值：<length> | <percentage> | auto | content
                <length>: 用长度值来定义宽度。不允许负值
                <percentage>: 用百分比来定义宽度。不用需负值
                auto ： 无特定宽值，取决于其他属性值，auto的计算

                规则是 检索一下你是否设置了width(或者height值，取决于flex-direction)就会采用这个值，否则的话最后采用的是content
                content: 基于内容自动计算宽度
        flex  复合属性
            flex:flex-grow,flex-shrink 和 flex-basis的简写，默认值为0 1 auto.后两个属性可选
            flex：none | [<'flex-grow'><'flex-shrink'>? || <'flex-basis'>]
            该属性有两个快捷值：auto(1 1 auto) 和 none(0 0 auto).
            建议优先使用这个属性，而不是单独写三个分离的属性，因为浏览器会推算相关值

Sass篇(速度篇)

1、定义变量，通过$定义变量，变量名可以带(-)中划线和(_)下划线,但是他们代表的值是一样的,如果重复定义变量名，则后边的变量覆盖前面的变量
2、支持css选择器进行嵌套，可以避免重复使用，主要可以：
    选择器嵌套;
    属性嵌套;
3、具有css的@import特性
    css的@import特性：允许在一个css文件中导入其他css,只有在加载到@import时，才会去下载，这会导致页面加载非常慢；
    sass的@import特性：sass在编译时，生成css文件就会把相关的文件引入进来，无需发起额外的下载请求，并且不需要指定引入文件的全名，并且sass的@import可以写道css的样式里面，若想在sass中引入css文件，则只需把css文件的后缀改为scss即可
    ex:
        @import "colors"---------->colors.scss
        @import "mixins"---------->mixins.scss
        @import "grid"------------>grid.scss
        .blue-theme {@import "blue-theme"}
4、Sass混合器
    类似js的函数
        不带参混合器声明和调用
            ex:
                @mixin mixName {        
                    float: left;
                    margin-left: 10px;
                }
                div {
                    @include mixName;
                }
                //编译后：
                div {
                float: left;
                margin-left: 10px;
                }
        带参数混合器的声明和调用
            ex:
                @mixin left($value: 10px) {
                    float: left;
                    margin-left: $value;
                }
                div {
                    @include left(66px);
                }
                //编译后：
                div {
                    float: left;
                    margin-left: 66px;
                }
        带多参数混合器的声明和调用
            ex:
            @mixin mixName($shadow...) {
                box-shadow:$shadow;
            }
            .box{
                 @include mixName(0 2px 2px rgba(0,0,0,.3),0 3px 3px rgba(0,0,0,.3),0 4px 4px rgba(0,0,0,.3));
            }
            //编译后：
            .box {
                 box-shadow: 0 2px 2px rgba(0, 0, 0, 0.3), 0 3px 3px rgba(0, 0, 0, 0.3), 0 4px 4px rgba(0, 0, 0, 0.3);
            }
5、Sass继承
    选择器继承可以让选择器继承另一选择器的所有样式，并联合声明，使用关键字@extends
        ex:
            .class1 {
                border: 1px solid #333;
            }
            .class2 {
                @extend .class1;
                background-color: #999;
            }
            //编译后
            .class1, .class2 {
                border: 1px solid #333;
            }
            .class2 {
                background-color: #999;
            }
